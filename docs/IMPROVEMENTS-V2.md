## 一、新增运维工具

### 1.1 日志查看和分析工具

**文件**：`scripts/log-viewer.ts`

**功能**：
- 查看最近日志（支持级别和类型过滤）
- 按时间范围查看日志
- 日志统计分析（按级别、类型、时间）
- 搜索日志（关键词搜索）
- 清理旧日志（保留指定天数）
- 导出日志（JSON 格式）
- 实时监控日志（Ctrl+C 退出）

**使用方式**：
```bash
pnpm logs
```

**特点**：
- 彩色输出，易于阅读
- 交互式菜单
- 支持多种过滤条件
- 实时监控模式

---

### 1.2 数据导入导出工具

**文件**：`scripts/data-export.ts`

**功能**：
- 导出用户数据（支持会员类型过滤）
- 导出生成记录（支持时间范围）
- 导出订单数据（支持状态过滤）
- 导出兑换码（支持状态过滤）
- 导出完整备份（所有数据）
- 导出为 CSV（Excel 兼容）
- 列出导出文件
- 清理导出文件

**使用方式**：
```bash
pnpm export
```

**导出格式**：
- JSON：完整数据结构
- CSV：Excel 可直接打开（UTF-8 BOM）

**导出目录**：`exports/`

**文件命名**：`{type}-{timestamp}.{json|csv}`

---

### 1.3 性能监控脚本

**文件**：`scripts/performance-monitor.ts`

**功能**：
- 系统信息（CPU、内存、运行时间）
- 数据库统计（数据量、今日新增）
- 性能指标（查询性能测试）
- 热点数据分析（活跃用户、热门类型）
- 收入分析（总收入、按产品统计、本月收入）
- 生成性能报告（JSON 格式）

**使用方式**：
```bash
pnpm monitor
```

**报告目录**：`reports/`

**监控指标**：
- 简单查询：< 50ms（绿色）
- 复杂查询：< 200ms（黄色）
- 慢查询：> 200ms（红色）

### 3.2 Git Hooks 系统

**文件**：`scripts/install-hooks.sh`

**安装的 Hooks**：

1. **pre-commit**
   - 运行 TypeScript 类型检查
   - 运行 ESLint 代码规范检查
   - 运行测试
   - 检测敏感信息
   - 依赖安全审计

2. **commit-msg**
   - 验证提交信息格式（Conventional Commits）
   - 支持的类型：feat、fix、docs、style、refactor、perf、test、chore、build、ci、revert
   - 格式：`<type>(<scope>): <subject>`

3. **pre-push**
   - 运行所有测试
   - 确保代码质量

**使用方式**：
```bash
# 安装 hooks
pnpm hooks:install

# 卸载 hooks
pnpm hooks:uninstall

# 跳过 hooks（不推荐）
git commit --no-verify
git push --no-verify
```

**配置文件**：`.githooks.json`

## 九、使用场景

### 9.1 日常开发

```bash
# 启动开发
pnpm dev

# 监听测试
pnpm test:watch

# 查看日志
pnpm logs
```

### 9.2 问题排查

```bash
# 1. 查看最近错误日志
pnpm logs
# 选择 "查看最近日志" → 选择 "error" 级别

# 2. 检查性能
pnpm monitor

# 3. 查看故障排查指南
cat docs/TROUBLESHOOTING.md
```

### 9.3 数据分析

```bash
# 导出用户数据
pnpm export
# 选择 "导出用户数据"

# 导出为 CSV（Excel 可打开）
pnpm export
# 选择 "导出为 CSV"

# 查看统计
pnpm tools:stats
```

### 9.4 代码提交

```bash
# 安装 Git Hooks（首次）
pnpm hooks:install

# 正常提交（自动运行检查）
git add .
git commit -m "feat(auth): 添加手机号登录"

# 推送（自动运行测试）
git push
```
